# Documentation Assets

This directory contains visual assets for Wheeler Memory documentation, organized into two categories.

## Diagrams (`diagrams/`)

Conceptual diagrams and animations illustrating core Wheeler Memory concepts:

- **evolution.gif**: Frame-by-frame CA evolution animation showing convergence to an attractor
- **phase_2_verification.png**: Verification results from Phase 2 development milestone

These assets support the markdown documentation in the parent `docs/` directory.

## Reports (`reports/`)

Test and validation reports documenting system performance and correctness:

- **diversity_report.png**: Attractor diversity validation (20 diverse inputs, correlation heatmap)
- **diversity_report_math.png**: Mathematical diversity metrics with CPU backend
- **diversity_report_math_gpu.png**: Diversity metrics with GPU acceleration (1000 samples)
- **diversity_report_math_10k_gpu.png**: Large-scale diversity validation (10,000 samples, GPU)
- **paraphrase_report.png**: Semantic similarity verification (fuzzy matching with embeddings)
- **paraphrase_embed_report.png**: Embedding-based recall validation
- **reconstruction_demo.png**: Reconstructive recall demonstration (blending and re-evolution)

These reports are generated by test scripts in `scripts/`:
- `test_diversity.py --output docs/assets/reports/diversity_report.png`
- `test_diversity_math.py --output docs/assets/reports/diversity_report_math.png`
- `test_paraphrase.py --output docs/assets/reports/paraphrase_report.png`
- `test_paraphrase_embed.py --output docs/assets/reports/paraphrase_embed_report.png`

## Organization Philosophy

- **Diagrams**: Illustrative, conceptual — explain the *why* and *how*
- **Reports**: Empirical, quantitative — demonstrate *correctness* and *performance*

This separation allows readers to quickly locate the type of visual information they need.
