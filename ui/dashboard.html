Wheeler Memory — Quick Guide

Store (Ctrl+Enter)
Saves your text as a memory. The system runs it through a cellular automaton — a self-organising grid — until a stable pattern forms. That pattern is the memory's unique fingerprint.
Recall (Shift+Enter)
Finds stored memories similar to your query by comparing their fingerprints using Pearson correlation. Higher similarity = closer match.
The colourful square
The attractor — the stable pattern the cellular automaton converged to. Think of it as the memory's visual fingerprint. Each input produces a unique one.
Temperature & Tier
How "fresh" this memory is. Starts at 0.3 when first stored, rises with repeated access, and halves every 7 days without use.
HOT ≥ 0.6 — recent or frequently accessed.
WARM ≥ 0.3 — moderately accessed.
COLD < 0.3 — stale, candidate for archival.
State
How the memory's pattern formed:
CONVERGED — settled into a stable pattern (good).
OSCILLATING — pattern kept cycling (uncertain).
CHAOTIC — never settled (rare, try rephrasing).
Ticks
How many steps the cellular automaton took to form this pattern. Typically 40–100 ticks (~3 ms on CPU).
Sleep
Compresses old memories by removing redundant intermediate frames, saving disk space. The first frame (seed) and last frame (attractor) are always kept. Hot memories are skipped.
Forget
Permanently deletes the selected memory — attractor, formation history, and index entry.
